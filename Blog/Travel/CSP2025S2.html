CSP2024前总结 & Day-1

为什么会在这里写 CSP2024 前总结？因为教练让写但是没地方放了。

Can You Tell Me Why WA

把 Luogu 主页的 Can You Tell Me Why WA 搬过来了。

  • unique() → 之前要 sort()

  • 多测清空 → 别用 memset() 用循环（指没有限制 T 限制 ∑n 的题目）

  • n≠m≠k≠q≠…

  • 不开 long long 见祖宗

    • #define int long long →MLEorTLE
    • int →WA
    • typedef long long ll; →AC

  • 不看 warning 见祖宗：

    • printf("%d\n", ans); warning: format '\%d' expects argument of type 'int', but argument 2 has type 'long long int' [-Wformat=]

  • n≤105→ 1ll * n * n

  • 头文件

    • 用万能头 → 不能用 y1 max
    • 不用万能头 → 把库写全啊

  • 数据结构

    • 传统线段树 → seg.Build(n);
    • 分块 → len = sqrt(n); br[Get(n)] = n;
    • 树剖 → Dfs2(...) { dfn[x] = ++dfc, ... }
    • 树状数组 → for (; x < N; x += x & -x) ... for (; x; x -= x & -x) ...

  • 找重儿子（树剖 & DSU on Tree）→ siz[x] = 1;

  • for (auto i : s) 中 auto 是一个大容器的时候要加 &，否则 TLE

  • (res += v) % P → WA (res += v) %= P → AC

  • std::cout << tmp << '\n'; → WA std::cerr << tmp << '\n'; → TLE // std::cerr << tmp << '\n'; → AC

  • char s[N], t[N], n, m; → RE char s[N], t[N]; int n, m; → AC

这基本就是一些比较唐的错误。

往 · 年 · 真 · 寄

然后想一下以前 CSP / NOIP 都是咋寄的。

  • HACSP2022J

    1. 没开 long long 好像，当时出考场突感想到要开但是忘了开没开，回去下载代码看见没开 long long 感觉真寄。
    2. 最短路好像写假了。

  • CSP2023S2

    1. 打表没删。（S虽o然r不a是t我o）
    2. 没写 n3 暴力暴力导致直接写 n3 DP 导致不会优化。
    3. 时间分配不合理导致大模拟没时间写部分分了。

  • NOIP2023

    1. 看错 T1 了导致写的特别麻烦（但是看错的竟然还是原题推论）
    2. T2 是并查集但是不会带权也没看出来带权。
    3. T4 O(2n) 的暴力挂了，性质也挂了，52→36。

  • 板子一定要保证正确。

  • 所以一定要看看freopen和打表删了，cerr 也删了注意别超时。

  • 注意答题策略（优化一个没有优化前途的暴力、没有看出来正解算法都很可怕）和时间分配。

  • 暴力出奇迹，对拍保平安。不可以，总司令！
